# \# 02. ДВИЖОК ДОВЕРИЯ (TRUST ENGINE)

# 

# \## 2.1. Радар Данбара (Геометрия независимых связей)

# 

# \* \*\*L1 (Информация)\*\*: Социальный граф пользователя расширен до 4 независимых функциональных кругов: \*\*Top 5\*\* (Ядро), \*\*15\*\* (Близкие), \*\*50\*\* (Друзья), \*\*150\*\* (Знакомые). Суммарная емкость активного графа — \*\*220 слотов\*\*.

# \* \*\*L2 (Пояснение)\*\*: В отличие от классической вложенной модели, здесь круги работают как независимые «корзины» (buckets). Это биологический предел эффективного социального взаимодействия. Рекомендации распределяются строго по этим орбитам, отсекая информационный шум и создавая персонализированный фильтр доверия.

# 

# \## 2.2. Наполнение Радара (Shadow List \& Radar UI)

# 

# \* \*\*L1 (Информация)\*\*: Используется механика \*\*Shadow List\*\* (Список ожидания) и визуальный интерфейс «Полурадар» в стиле \*\*Glassmorphism\*\*.

# \* \*\*L2 (Пояснение)\*\*: Новые контакты попадают в горизонтальный «серый список» под радаром. Распределение в круги идет через Drag-and-drop на 4 концентрические полуокружности. Интерфейс выполнен в нативной палитре Telegram (Морская волна/Teal) с убывающей интенсивностью цвета от центра. Доступна механика JIT (Just-In-Time) и свайп вниз для удаления неактуальных связей.

# 

# \## 2.3. Матрица ABCD

# 

# \* \*\*L1 (Информация)\*\*: Детальная оценка мастера состоит из 4 параметров (от 1 до 5): \*\*A\*\* (Able/Мастерство), \*\*B\*\* (Believable/Честность), \*\*C\*\* (Connected/Сервис), \*\*D\*\* (Dependable/Надежность).

# \* \*\*L2 (Пояснение)\*\*: Клиент видит одну цифру (Score), но при клике на нее открывается лепестковая диаграмма ABCD. Это позволяет понять детали: мастер может делать работу блестяще (A=5), но быть необязательным в сроках (D=2).

# 

# \## 2.4. Социальный Арбитраж

# 

# \* \*\*L1 (Информация)\*\*: Трехуровневая система разрешения споров: Презумпция факта -> Публичный ответ -> Платная Апелляция.

# \* \*\*L2 (Пояснение)\*\*: Если мастер не согласен с отзывом, он вносит залог (Stake) ~$10 в ⭐️ и предоставляет доказательства. Админ выносит вердикт. При победе мастера отзыв удаляется, залог возвращается. При проигрыше — залог сгорает в пользу платформы, отзыв получает статус «Подтверждено Арбитром».

# 

# \## 2.5. Математика Данбар-Рейтинга

# 

# \* \*\*L1 (Информация)\*\*: Данбар-рейтинг вычисляется индивидуально для каждого клиента на основе средневзвешенной оценки его кругов доверия.

# \* \*\*L2 (Пояснение)\*\*: Весовые коэффициенты ($w$) адаптированы под систему 4-х кругов:

# \* \*\*Top 5\*\*: $w = 2.0$

# \* \*\*Circle 15\*\*: $w = 1.5$

# \* \*\*Circle 50\*\*: $w = 1.2$

# \* \*\*Circle 150\*\*: $w = 1.0$

# \* \*\*Global (Вне кругов)\*\*: $w = 0.5$

# 

# 

# 

# Формула расчета:

# 

# 

# $$R\_{dunbar} = \\frac{\\sum\_{i=1}^{n} (r\_i \\cdot w\_i)}{\\sum\_{i=1}^{n} w\_i}$$

# 

# \## 2.6. Разделение рейтингов (Public vs Personal)

# 

# \* \*\*L1 (Информация)\*\*: Система разделяет глобальную популярность мастера и его личную ценность для конкретного пользователя.

# \* \*\*L2 (Пояснение)\*\*:

# \* \*\*Public Global Score\*\*: "Витринный" рейтинг, где все голоса считаются с базовым весом $w=0.5$. Показывает общую рыночную адекватность.

# \* \*\*Personal Dunbar Score\*\*: Приоритетный рейтинг. Если в оценке мастера участвовали люди из кругов пользователя, их мнение доминирует в выдаче. Если личных связей нет, Personal Score приравнивается к Global Score.
