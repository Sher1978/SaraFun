# 07. ПАНЕЛЬ АДМИНИСТРАТОРА (BACKOFFICE)

## 7.1. Финансовый контроллер
* **L1 (Информация)**: Раздел управления глобальной экономикой платформы.
* **L2 (Пояснение)**: Админ видит общий пул ликвидности (сколько ⭐️ заведено в систему). Здесь происходит ручной аппрув заявок на вывод в TON. Также здесь находится ползунок изменения глобальных параметров: размер абонплаты (по умолчанию $5), % комиссии платформы, ручная корректировка курса Оракула (при сбоях API Fragment).

## 7.2. Арбитражный стол (Disputes)
* **L1 (Информация)**: Рабочее место модератора для разбора конфликтных отзывов.
* **L2 (Пояснение)**: Сюда падают заявки от мастеров, оплативших залог (Stake). Админ видит отзыв, пруфы мастера (фото/чат) и выносит вердикт: "Удалить отзыв" (залог возвращается) или "Оставить отзыв" (залог переходит на счет платформы).

## 7.3. Управление пользователями и модерация
* **L1 (Информация)**: CRM-система базы SaraFun.
* **L2 (Пояснение)**: Возможность найти любого пользователя по `tg_uid`. Функции: теневой бан (Shadowban) за мошенничество, принудительное скрытие непристойных фото в карточках услуг, просмотр дерева рефералов пользователя.

## 7.4. Аналитика (Dashboard)
* **L1 (Информация)**: Ключевые метрики здоровья продукта (DAU, MAU, GMV).
* **L2 (Пояснение)**: Графики: объем транзакций в $ за сутки, количество новых регистраций, средний чек по 7 секторам, количество отмененных сделок (60-sec Undo).

## 7.5. Superadmin Dashboard (Web Panel)
* **L1 (Информация)**: Полноценная веб-панель управления платформой со специализированными журналами (Logs).
* **L2 (Пояснение)**: Доступна в Telegram WebApp, оптимизирована для десктопов (ПК), но масштабируется для смартфонов в виде стопки карточек. Включает следующие вкладки:
    * **Системные параметры (Settings)**: Управление глобальными переменными (Visibility fee, комиссии платформы, лимиты транзакций).
    * **Журнал пользователей (Users Log)**: Таблица всех пользователей с кнопкой "Детали" для детальной разбивки данных (баланс, статус онбординга, ограничения, профиль мастера).
    * **Журнал транзакций (Transactions Log)**: Детальная база данных транзакций платформы со статусами разделения средств (Split math).
    * **Журнал рефералов (Referrals Log)**: Дерево связей (кто кого пригласил и сколько заработал) для отслеживания фрода и MLM-расчетов.
    * **Управление ролями (Roles)**: Узел контроля доступа. Суперадмин может назначать другим пользователям 3 уровня привилегий: "Суперадмин" (полный доступ), "Админ" (всё кроме распределения ролей), "Менеджер" (только просмотр и модерация споров).
